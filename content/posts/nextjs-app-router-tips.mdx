## 1. Server Components 預設，Client Components 按需

App Router 的所有元件預設都是 Server Component，只有需要互動（`useState`、`useEffect`、事件處理）才加 `"use client"`。

```tsx
// ✅ 這樣就好，不需要 "use client"
export default async function PostList() {
  const posts = await fetchPosts(); // 直接在 server 呼叫
  return <ul>{posts.map(p => <li key={p.id}>{p.title}</li>)}</ul>;
}
```

## 2. 善用 `loading.tsx`

在每個 route segment 放一個 `loading.tsx`，Next.js 會自動用 Suspense 包起來：

```
app/
  blog/
    loading.tsx   ← 這個
    page.tsx
```

```tsx
export default function Loading() {
  return <div className="animate-pulse">載入中...</div>;
}
```

## 3. `generateMetadata` 處理動態 SEO

```tsx
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const post = await getPost(params.slug);
  return {
    title: post.title,
    description: post.summary,
    openGraph: { title: post.title, description: post.summary },
  };
}
```

## 4. Route Groups 整理路由結構

用 `(folder)` 括號命名的資料夾不會影響 URL 路徑，但可以讓你共用 layout：

```
app/
  (marketing)/
    layout.tsx   ← 行銷頁共用 layout
    page.tsx
    about/
  (app)/
    layout.tsx   ← 應用頁共用 layout（有側邊欄）
    dashboard/
    settings/
```

## 5. `unstable_cache` 快取資料

```tsx
import { unstable_cache } from "next/cache";

const getCachedPosts = unstable_cache(
  async () => fetchPostsFromDB(),
  ["posts"],
  { revalidate: 3600 } // 1 小時後重新驗證
);
```
